table_extractor › 全体処理フロー（Step 1〜7） 各ステップの入出力のつながりのみを示す俯瞰図です / カードをクリックすると各ステップの詳細フローへ遷移できます
入力
アップロードされた Excel / CSV ファイル
bytes（ファイル内容）、ファイル名
出力
エクスポート用 ZIP（CSV 一式）
選択済みテーブルの CSV + 監査用メタデータ JSON
STEP 1 📄 ファイル読み込みモジュール 詳細フローを見る →
ファイルのバイト列を受け取り、セル値を保持した生グリッド（1-indexed 2次元配列）を構築する。 数式評価・結合セル伝播を行う。テーブル検出・整形・分析は行わない。
入力
bytes（Excel / CSV）、ファイル名
出力
List[SheetGrid] / pd.DataFrame（CSV）
List[SheetGrid]（シートごとの生グリッド）／ pd.DataFrame
STEP 2 🔍 テーブル検出モジュール 詳細フローを見る →
生グリッドまたは DataFrame を受け取り、行分類ステートマシンでテーブル領域を特定する。 ファイル読み込み・整形・分析は行わない。
入力
List[SheetGrid] / pd.DataFrame、ファイル名
出力
List[DetectedTable]（位置・DataFrame・タイトル・注記）
List[DetectedTable]（各テーブルの生 DataFrame を内包）
STEP 3 🧹 テーブル構造正規化モジュール 詳細フローを見る →
検出された生 DataFrame を分析に適した形式に正規化する。多段ヘッダーの統合・Pivot（属性/値ペアの昇格） 検出・Transpose（行列逆転）検出・多軸ヘッダー展開・グルーピング列の補完・「うち」書きの内訳分離・ 集計行列の除去・単位混在の分離・無効カラム（全欠損列・無名列）の削除・クロス集計/Wide_to_longの 縦持ち変換・ファイル外メタデータ（ファイル名・シート名由来の付帯情報）の派生カラム生成を行う。 決定論的処理（step3_normalize_determ.py）とLLM使用処理（step3_normalize_llm.py）に分離されている。
入力
DetectedTable.df（step2 が構築した生 DataFrame）
出力
正規化済み DataFrame + 整形メタ情報（DetectedTable に付与）
List[DetectedTable]（正規化済み）
STEP 4 🤖 テーブル分析モジュール 詳細フローを見る →
正規化済みテーブルリストを OpenAI API に送信し、テーブルの意味・関係を分析する。API 呼び出し前に ルールベースで合計関係・シート階層を事前計算してプロンプトに付加する。統合推奨も同一 API 呼び出しで 生成し結果に含める（提示は step5 が担う）。
入力
List[DetectedTable]（正規化済み）、ファイル名
出力
AIAnalysisResult（分類・分析・統合推奨・マスタ情報・要約）
AIAnalysisResult
STEP 5 💡 新規テーブル提案モジュール 詳細フローを見る →
テーブル関係分析（step4）の結果を受けて、統合・潜在・導出の3種類の新規テーブル提案を集約する出口。 潜在テーブル提案は注記・注釈からルールベースで示唆し、導出テーブル提案は親−子の数値差分から算出する （いずれも API 呼び出しなし）。
入力
AIAnalysisResult、List[DetectedTable]（正規化済み）
出力
List[LatentTableProposal] / List[DerivedLatentTable]
AIAnalysisResult ＋ 潜在／導出テーブル提案
（UI 層のユーザー決定を経て final_tables 統合済みテーブル辞書を生成）
STEP 6 ✅ テーブル選択モジュール 詳細フローを見る →
AI 分析結果と統合推奨をもとに、エクスポート対象テーブルのグルーピング・粒度バッジ生成・ ファイル名変換を行う。
入力
AIAnalysisResult、final_tables（統合済みテーブル辞書）
出力
グループ分類辞書、バッジ HTML、安全なファイル名
選択済みテーブル Dict[str, dict]（df・display_name 等）
STEP 7 📦 エクスポートモジュール 詳細フローを見る →
選択済みテーブルを CSV に変換し、ZIP アーカイブとして梱包する。集計除去の監査用メタデータが 付与されたテーブルは同名の _metadata.json も同梱する。
入力
Dict[str, dict]（選択済みテーブル）
出力
ZIP バイト列、Dict[str, bytes]（ファイル名→バイト列）
▲ 各カードは対応するモジュール（src/step1_upload.py 〜 step7_export.py）の docstring に基づく入出力の要約です。
各カードをクリックすると、そのステップの関数単位の詳細フローに遷移します。

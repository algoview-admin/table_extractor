Transpose検出と変換機能

東京10020
大阪8015

「うち」書き識別と別テーブル分離機能

「うち男性」等の内訳行を検出し、親子関係を保ったまま別テーブルへ分離する

テーブルサンプル
変換前
支店区分人数
東京社員100
東京うち男性60
東京うち女性40
▼分離
変換後（メイン）
支店区分人数
東京社員100
＋新規
内訳テーブル（新規）
支店親区分子区分人数
東京社員男性60
東京社員女性40

2024人口1200
2024労働力率62.5
＋新規
指標マスタ（新規）
指標単位
人口人

Wide_to_long検出と変換機能

軸（時系列軸・支店軸等）と指標名が1つの列名に合成された表を検出し、軸について縦持ちに変換する（指標は列として維持する）。
決定論的な手段（時系列語彙・区切り文字）を優先し、それでも判定できない場合のみLLMで確認する。

テーブルサンプル
① 時系列語彙（決定論的・LLM不使用）
変換前（横持ち）
支店2023_売上2023_原価2024_売上2024_原価
東京1006012065
▼縦持ち化
変換後（縦持ち）
支店年売上原価
東京202310060
東京202412065
② 区切り文字あり（決定論的・LLM不使用）
変換前（横持ち）
商品東京支社_売上東京支社_原価大阪支社_売上大阪支社_原価名古屋支社_売上名古屋支社_原価
家電1007080506040
▼縦持ち化
変換後（縦持ち）
商品支社売上原価
家電東京支社10070
家電大阪支社8050
家電名古屋支社6040
軸トークン共通の接尾辞「支社」を軸名に採用（共通部分がなければ「区分」）
③ 区切り文字なし（LLM 使用）

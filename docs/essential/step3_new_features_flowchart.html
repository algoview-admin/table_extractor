無効カラムの検出と削除機能

全欠損列・無名列（プレースホルダ列名）を検出し、全欠損列は既定で削除する（ユーザーがチェックボックスで復元・追加削除を調整可能）

テーブルサンプル
変換前
支店値Unnamed: 3
東京参考A
大阪
▼既定削除
変換後
支店Unnamed: 3
東京参考A
大阪
値は列名は正式だが全欠損のため既定で削除（reason: 全欠損）／3列目は列名も空欄（無名）かつ全欠損のため既定で削除（reason: 無名＋全欠損）／Unnamed: 3は無名だがデータがあるため既定では保持（reason: 無名（データあり）、候補として提示のみ）

ファイル外メタデータからの派生カラム生成機能

ファイル名・シート名から表本体に現れない付帯情報をLLMで抽出し、既存列のどの列と関連が深いか（前後どちらに置くべきか）を判定し、その位置に定数列として埋め込む

入力元（表本体には現れない情報）

・シート名: 支店別（契約数）
▼情報抽出
変換前（表本体）
区分14月5月
ライトプラン2022
▼列挿入位置判定
変換後（抽出列情報を関連する列の前後に定数列として埋め込み）
サービス名区分1指標名年度4月5月
らくらく管理ライトプラン契約数20252022
サービス名はファイル名の「らくらく管理」から抽出し区分1の前（上位区分）に、指標名はシート名の「（契約数）」から抽出し区分1の後（値の種類）に、年度はファイル名の「2025年度」から抽出し集計軸（月）の直前に挿入
